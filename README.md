# Vector Search 기반 콘텐츠 추천 데모

> 막연한 사용자 요구를 **Embedding → Vector Search → Top-K Retrieval → 조건 기반 재정렬** 흐름으로 연결하는 청중 참여형 의미 기반 검색 데모입니다.

이 프로젝트는 사용자가 입력한 단어나 문장을 벡터로 변환하고, 검색어와 의미적으로 가까운 항목을 찾아 워드클라우드에서 강조하는 방식으로 Vector Search의 작동 원리를 보여줍니다.

발표 주제는 **“느낌을 검색하는 방법: Vector Search로 찾아내는 사용자의 의도”**입니다.

---

## 1. 프로젝트 개요

우리는 유튜브, 넷플릭스, 쇼츠, 릴스 같은 콘텐츠 플랫폼에서 추천 기술을 자연스럽게 경험합니다. 사용자는 정확한 제목을 검색하지 않아도, 현재 관심사나 시청 상황에 맞는 콘텐츠를 추천받습니다.

이 프로젝트의 핵심 질문은 다음과 같습니다.

> 컴퓨터는 사용자의 막연한 표현이나 느낌을 어떻게 계산할 수 있을까?

예를 들어 다음 두 표현은 서로 다른 단어를 사용합니다.

```text
밥 먹을 때 가볍게 볼 영상
혼밥할 때 보기 좋은 짧은 음식 리뷰
```

사람은 두 표현이 비슷한 상황을 가리킨다는 것을 쉽게 이해합니다. 이 프로젝트는 이러한 사람의 직관적인 “비슷함”을 컴퓨터가 계산 가능한 벡터 유사도 문제로 바꾸는 과정을 시각적으로 보여줍니다.

---

## 2. 핵심 아이디어

Vector Search는 텍스트를 숫자 벡터로 변환한 뒤, 벡터 공간에서 의미적으로 가까운 항목을 찾는 검색 방식입니다.

```text
사용자 입력
   ↓
Embedding
   ↓
Vector Representation
   ↓
Cosine Similarity
   ↓
Vector Search
   ↓
Top-K Retrieval
   ↓
워드클라우드 강조
```

이 흐름을 통해 검색어와 데이터가 정확히 같은 문자열을 공유하지 않아도, 의미적으로 가까운 결과를 찾을 수 있습니다.

---

## 3. 주요 기능

### 청중 입력 수집

청중은 QR을 통해 입력 화면에 접속한 뒤, 떠오르는 단어나 짧은 문구를 자유롭게 입력합니다.

입력 예시는 다음과 같습니다.

```text
먹방
공포
여행
힐링
잠들기 전
편하게 볼 것
짧은 리뷰
예능
```

입력된 값은 기본 검증과 전처리를 거쳐 저장됩니다.

```text
청중 입력
→ 공백 제거
→ 길이 제한
→ 중복 확인
→ 저장
→ 워드클라우드 반영
```

### 워드클라우드 시각화

입력된 단어와 문구는 발표자 화면의 워드클라우드에 표시됩니다.

일반적인 워드클라우드는 단어 빈도를 중심으로 시각화됩니다. 이 프로젝트에서는 검색어와 의미적으로 가까운 Top-K 단어를 강조하여 Vector Search 결과를 시각적으로 보여줍니다.

### 검색어 기반 Top-K 검색

발표자는 검색어와 K 값을 입력합니다.

예를 들어 검색어가 다음과 같다고 가정할 수 있습니다.

```text
편하게 볼 것
```

이때 시스템은 청중이 입력한 단어 중 검색어와 의미적으로 가까운 상위 K개를 찾습니다.

예상 결과 예시는 다음과 같습니다.

```text
힐링
예능
먹방
잠들기 전
짧은 리뷰
```

정확히 같은 표현이 입력되지 않아도, 의미적으로 가까운 항목이 강조될 수 있습니다.

## 4. 기술 흐름

### 4-1. Embedding

Embedding은 자연어 텍스트를 숫자 벡터로 변환하는 과정입니다.

예를 들어 “고양이”라는 텍스트는 임베딩 모델을 통해 다음과 같은 벡터로 변환될 수 있습니다.

```text
고양이
→ [0.12, -0.35, 0.81, ...]
```

이번 데모에서 사용하는 임베딩 모델은 텍스트 하나를 고정된 차원의 벡터로 변환합니다. 짧은 단어와 긴 문장 모두 같은 차원의 벡터로 표현되기 때문에, 컴퓨터는 서로 길이가 다른 텍스트도 같은 기준에서 비교할 수 있습니다.

### 4-2. Vector Search

Vector Search는 텍스트 또는 문서를 벡터로 변환한 뒤, 벡터 공간에서 가까운 벡터를 찾는 검색 방식입니다.

```text
검색어
→ Embedding
→ query_vector

청중 입력값
→ Embedding
→ word_vectors

query_vector와 word_vectors 비교
→ 의미적으로 가까운 Top-K 항목 반환
```

### 4-3. Cosine Similarity

Cosine Similarity는 두 벡터의 방향이 얼마나 비슷한지 계산하는 방식입니다.

```text
cosine_similarity(query_vector, content_vector)
→ 검색어와 입력값의 의미적 가까움
```

두 벡터가 비슷한 방향을 향할수록 의미적으로 가까운 것으로 볼 수 있습니다.

### 4-4. Top-K Retrieval

Top-K Retrieval은 유사도 점수가 높은 상위 K개 결과를 반환하는 과정입니다.

```text
검색어 벡터
→ 전체 입력 벡터와 유사도 계산
→ 유사도 점수 정렬
→ 상위 K개 결과 반환
```

예를 들어 K가 5라면, 검색어와 가장 가까운 5개의 단어 또는 문구가 반환됩니다.

## 5. Azure 기반 구조

이 데모는 Azure 서비스를 활용하여 의미 기반 검색 흐름을 구성할 수 있습니다.

### Azure OpenAI

Azure OpenAI는 텍스트를 임베딩 벡터로 변환하는 역할을 합니다.

```text
텍스트 입력
→ Azure OpenAI Embedding Model
→ Embedding Vector
```

### Azure AI Search

Azure AI Search는 벡터화된 데이터를 인덱싱하고, 검색어 벡터와 가까운 항목을 찾는 역할을 합니다.

```text
청중 입력 벡터 저장
→ Vector Index 구성
→ 검색어 벡터와 비교
→ Top-K 결과 반환
```

전체 구조는 다음과 같습니다.

```text
청중 입력 텍스트
→ Azure OpenAI Embedding
→ Azure AI Search Index 저장

발표자 검색어
→ Azure OpenAI Embedding
→ Query Vector 생성
→ Azure AI Search Vector Search
→ Top-K 결과 반환
→ 워드클라우드 강조
```

## 6. 데모 시나리오

### Step 1. QR 접속

청중은 QR 코드를 통해 입력 화면에 접속합니다.

### Step 2. 단어 또는 문구 입력

청중은 자신이 떠올린 콘텐츠 키워드, 분위기, 장르, 관심사를 자유롭게 입력합니다.

```text
먹방
공포
축구
여행
잠들기 전
가볍게 볼 것
```

### Step 3. 워드클라우드 표시

입력된 값이 발표자 화면의 워드클라우드에 반영됩니다.

### Step 4. 발표자 검색어 입력

발표자는 검색어와 K 값을 입력합니다.

```text
검색어: 편하게 볼 것
K: 5
```

### Step 5. Top-K 결과 강조

검색어와 의미적으로 가까운 상위 K개 항목이 워드클라우드에서 강조됩니다.

### Step 6. 결과 해석

청중은 검색어와 정확히 같은 문자열이 없어도 의미적으로 가까운 단어가 검색될 수 있다는 점을 확인할 수 있습니다.

## 7. Keyword Search와 Vector Search 비교

| 구분 | Keyword Search | Vector Search |
|---|---|---|
| 검색 기준 | 문자열 직접 일치 | 의미적 유사성 |
| 입력 방식 | 짧은 검색어 중심 | 단어, 문장, 맥락 중심 |
| 강점 | 명확한 제목·고유명사 검색에 적합 | 표현이 달라도 관련 의미 검색 가능 |
| 한계 | 표현이 다르면 결과를 놓칠 수 있음 | 임베딩 품질과 데이터 구성에 영향을 받음 |
| 적합한 상황 | 영화 제목, 인물명, 특정 키워드 검색 | 분위기, 상황, 취향 기반 검색 |

## 8. 조건 기반 재정렬

Vector Search로 반환된 결과를 그대로 최종 추천 결과로 사용할 수도 있습니다. 추천 서비스로 확장할 경우 사용자의 선호 조건과 회피 조건을 반영해 최종 순위를 조정할 수 있습니다.

예시는 다음과 같습니다.

```text
선호 조건:
- 짧은 영상
- 가벼운 분위기
- 먹방
- 예능

회피 조건:
- 공포
- 긴 다큐
- 무거운 분위기
```

최종 점수는 다음 흐름으로 계산할 수 있습니다.

```text
final_score
= vector_similarity
+ preference_bonus
- negative_penalty
- duration_penalty
```

이 구조를 적용하면 의미적으로 가까운 후보를 먼저 찾은 뒤, 사용자의 현재 기호에 맞게 최종 추천 결과를 조정할 수 있습니다.

## 9. 확장 아이디어: Catnap

Catnap은 Vector Search 원리를 콘텐츠 추천 서비스 경험으로 확장한 아이디어입니다.

사용자는 검색어를 길게 입력하지 않고, 현재 상황과 취향을 카드로 선택합니다.

```text
밥친구 모드
잠들기 전 모드
짧게 웃고 싶음 모드
가볍게 틀어놓기 모드
집중 없이 보기 모드
```

사용자의 선택값은 사용자 요구 프로필로 변환됩니다.

```text
상황: 밥친구 모드
선호: 먹방, 짧은 영상, 가벼운 분위기, 예능형 콘텐츠
회피: 공포, 긴 다큐, 무거운 분위기
```

이후 시스템은 다음과 같은 문장형 요구 프로필을 생성할 수 있습니다.

```text
사용자는 밥을 먹으면서 부담 없이 볼 수 있는 짧고 가벼운 콘텐츠를 원합니다.
먹방이나 예능형 콘텐츠에 긍정적이며, 공포나 긴 다큐처럼 몰입 부담이 큰 콘텐츠는 피하고 싶어 합니다.
```

이 요구 프로필을 Embedding하고, 콘텐츠 설명문 벡터와 비교하면 사용자의 현재 상황에 맞는 콘텐츠 후보를 찾을 수 있습니다.

## 10. 프로젝트 의의

이 프로젝트는 Vector Search를 복잡한 추천 시스템 전체로 설명하기보다, 청중이 직접 입력하고 결과 변화를 확인할 수 있는 작은 데모로 보여주는 데 초점을 둡니다.

핵심 의의는 다음과 같습니다.

- 사용자의 막연한 표현을 검색 가능한 데이터로 변환합니다.
- 텍스트를 벡터로 바꾸어 의미적 유사도를 계산합니다.
- Top-K 검색 결과를 워드클라우드에서 직관적으로 확인할 수 있습니다.
- Vector Search가 콘텐츠 추천 경험으로 확장될 수 있음을 보여줍니다.

## 11. 보안 및 운영 주의사항

실제 배포 또는 발표 환경에서는 다음 정보를 공개 저장소에 포함하지 않아야 합니다.

```text
API Key
Azure Endpoint
Connection String
실행 로그
발표 중 수집된 원본 입력 데이터
```

환경변수는 `.env.local` 또는 배포 플랫폼의 Secret 설정을 통해 관리해야 합니다.

예시는 다음과 같습니다.

```env
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_EMBEDDING_DEPLOYMENT=
AZURE_AI_SEARCH_ENDPOINT=
AZURE_AI_SEARCH_API_KEY=
AZURE_AI_SEARCH_INDEX_NAME=
```

공개 저장소에는 `.env.example`만 포함하고, 실제 인증 정보가 담긴 `.env.local`은 `.gitignore`에 추가해야 합니다.

## 13. 정리

Vector Search의 핵심은 사람의 직관적인 “비슷함”을 컴퓨터가 계산할 수 있는 벡터 유사도 문제로 바꾸는 것입니다.

```text
사람의 느낌
→ 텍스트
→ 임베딩 벡터
→ 유사도 계산
→ 의미 기반 검색
→ Top-K 결과
```

이 프로젝트는 청중 참여형 워드클라우드 데모를 통해 이 흐름을 시각적으로 보여줍니다. 검색어와 데이터가 정확히 일치하지 않아도 의미적으로 관련 있는 결과를 찾을 수 있다는 점이 이 프로젝트의 핵심입니다.
