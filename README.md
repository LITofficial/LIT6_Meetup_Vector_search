# 설명 가능한 벡터 검색 기반 컨텐츠 추천 알고리즘

---

# 프로젝트 구조

- MealTube(가명): 설명 가능한 벡터 검색 기반 콘텐츠 추천 시스템
    
    → 사용자가 유튜브 추천이나 검색 과정에서 영상을 바로 고르지 못할 때, 직접 키워드 카드를 선택해 현재 취향을 명시적으로 입력하고, 이를 벡터 검색 질의로 변환하여 의미적으로 가까운 콘텐츠를 추천하는 서비스
    

---

## 1. 문제 인식

- 유튜브, 넷플릭스, 쇼츠, 릴스처럼 콘텐츠 플랫폼은 이미 일상적으로 사용되고 있다.
- 콘텐츠의 양은 많아졌지만, 사용자는 오히려 “뭘 봐야 하지?”라는 선택 피로를 자주 겪는다(특히 밥을 먹을 때, 쉬는 시간, 이동 중처럼 가볍게 콘텐츠를 보고 싶은 상황).
- 위와 같은 상황에서 사용자는 막연한 요구만을 가지고 있을 수 있다(ex. “밥 먹을 때 가볍게 볼 영상”, “짧고 재밌는 콘텐츠”, “지금 기분에 맞는 영상” 등). ⇒ 직접 검색어를 입력하기엔 난감하다.
- 그렇다면 사용자는 홈 화면에서, 즉 알고리즘의 추천에서 마음에 드는 영상을 찾지 못하면 계속 스크롤을 내리거나, 새로고침을 해야만 한다.
- 이런 상황에서, 사용자는 알고리즘의 선택에 몸을 맡길 수 밖에 없는가? 소중한 시간을? 언제까지?

***→ 어떻게 하면 “사용자의 막연한 요구”를 구체화하고, 그 요구가 실제 추천 결과에 반영되도록 하여 더 빠르고 직관적으로 콘텐츠를 고를 수 있게 할 수 있을까?***

## 2. 기존 유튜브 추천 방식과 MealTube의 차이

**2-1. 기존 유튜브 추천 방식**

- 기존 유튜브 알고리즘은 사용자의 과거 시청 기록, 검색 기록, 클릭 기록, 구독 채널, 영상 반응 등을 바탕으로 관심 있을 만한 콘텐츠를 자동으로 제공한다.
- 사용자는 홈 화면에 나타난 영상의 썸네일과 제목을 보고 영상을 선택한다.
- 마음에 드는 영상이 보이지 않으면 스크롤을 계속 내리거나, 새로고침을 하거나, 직접 검색어를 입력해야 한다.
- 이 방식은 편리하지만, 사용자가 지금 당장 원하는 방향으로 알고리즘에 개입할 수는 없다.
- 추가로 이 방식은 특정 영상이 왜 추천되었는지, 어떤 기준이 강하게 반영되었는지 사용자가 확인하기 어렵다. 즉, 블랙박스형 추천의 형태를 가지고 있다.

**2-2. 블랙박스형 추천의 한계**

- 블랙박스형 추천은 입력과 출력은 확인할 수 있지만, 중간 판단 과정이 사용자에게 명확하게 드러나지 않는 추천 방식을 의미한다.
- 사용자는 추천 결과를 확인할 수는 있지만, 해당 영상이 최근 시청 기록, 유사 사용자 행동, 특정 키워드, 채널 선호도 중 어떤 기준으로 추천되었는지 알기 어렵다.
- 이로 인해 사용자는 자신이 원하는 방향을 추천 과정에 직접 반영하지 못한 채, 이미 제시된 영상 목록 안에서 마음에 드는 영상을 찾아야 한다.
- 특히 현재 기분이나 상황에 맞는 영상을 찾고 싶은 경우, 과거 기록 중심의 추천만으로는 사용자의 즉각적인 요구를 충분히 반영하기 어렵다.

**2-3. MealTube의 방향성**

- MealTube는 사용자가 유튜브 홈 추천이나 검색 과정에서 영상을 바로 고르지 못하는 순간을 보완하는 프로그램/사이트이다.
- 사용자가 홈 화면에서 원하는 영상을 찾지 못하거나, 검색어를 떠올리기 어려울 때 MealTube를 활용할 수 있다.
- MealTube는 사용자가 직접 키워드 카드를 선택하면서 현재 보고 싶은 콘텐츠의 방향을 선택하도록 한다.
- 이 선택 결과는 검색 질의를 구성하는 입력 데이터로 활용된다.
- 시스템은 사용자의 선택을 취향 프로필로 변환하고, 이를 벡터 검색에 활용하여 의미적으로 가까운 콘텐츠를 추천한다.
- 추천 결과에는 어떤 키워드가 반영되었는지, 어떤 콘텐츠 태그와 의미적으로 가까운지, 왜 해당 영상이 추천되었는지 등 자세한 정보를 함께 표시할 수 있다.
- 즉, MealTube는 사용자가 자신의 취향을 알고리즘에만 맡기는 것이 아니라, 원하는 방향으로 직접 개입하고 그 결과를 확인할 수 있도록 돕는다.
- 이를 통해 사용자는 자신의 취향에 맞는 선택을 더 빠르고, 더 직관적으로 할 수 있게 된다.

**2-4. 기존 방식과 MealTube 비교**

| 구분 | 기존 유튜브 추천 | MealTube |
| --- | --- | --- |
| 기본 방식 | 과거 시청 기록, 클릭 기록, 검색 기록 등을 바탕으로 자동 추천 | 사용자가 직접 선택한 키워드 카드를 바탕으로 추천 |
| 알고리즘 성격 | 추천 이유가 잘 보이지 않는 블랙박스형 추천에 가까움 | 선택 키워드, 관련 태그, 유사도를 확인할 수 있는 설명 가능한 추천 |
| 사용자 역할 | 홈 화면 추천 목록을 보고 선택 | 추천 전 단계에서 직접 취향 입력 |
| 선택 방식 | 썸네일·제목 확인, 스크롤, 새로고침, 직접 검색 | 키워드와 이미지가 포함된 카드 선택 |
| 검색 방식 | 직접 검색어 입력 또는 홈 화면 알고리즘 추천 | 선택 데이터를 취향 프로필로 변환한 뒤 벡터 검색 수행 |
| 추천 근거 | 사용자가 정확히 알기 어려움 | 선택 키워드, 관련 태그, 유사도 기반 근거 확인 가능 |
| 사용자 경험 | 알고리즘이 제시한 결과 중에서 고르는 방식 | 사용자가 현재 취향을 먼저 입력하고 그에 맞는 결과를 받는 방식 |
| 프로젝트 방향성 | 자동 추천 중심 | 사용자가 개입하는 추천, 설명 가능한 추천, 현재 취향 반영 |

## 3. MealTube와 핵심 아이디어

**3-1. MealTube란?**

- MealTube는 사용자가 카드 몇 장을 선택하면, 그 선택 결과를 검색 질의로 변환하고 벡터 검색을 통해 의미적으로 가까운 영상을 추천하는 서비스이다.
- 출발점은 “밥 먹을 때 뭐 보지?”, “유튜브 켰는데 볼 게 없네”, “넷플릭스에서 고르다가 시간 다 갔다”와 같은 일상적인 상황이다.
- 사용자는 직접 긴 검색어를 입력하지 않고, `먹방`, `예능`, `짧은 영상`, `공포`, `긴 다큐`, `AI`, `경제`, `여행` 같은 키워드 카드를 선택한다.
- 시스템은 이 선택 결과를 바탕으로 사용자의 현재 취향을 해석하고, 이를 검색 가능한 취향 프로필로 변환한다.
- 이후 취향 프로필을 임베딩하여 벡터 검색 질의로 만들고, 콘텐츠 인덱스에 저장된 영상 벡터와 비교하여 추천 결과를 반환한다.
- 추천 결과에는 단순히 영상 목록만 표시하는 것이 아니라, 선택 키워드와 관련 태그, 추천 근거까지 함께 제공한다.

**3-2. 왜 카드를 선택하는가**

- 카드 선택은 MealTube의 핵심 기술이라기보다, 사용자의 현재 취향을 쉽게 입력받기 위한 인터페이스이다.
- 사용자는 자신이 보고 싶은 콘텐츠를 매번 문장이나 검색어로 정확히 표현하기 어렵다. 특히 “밥 먹을 때 가볍게 볼 영상”, “짧고 재밌는 콘텐츠”처럼 요구가 막연한 경우에는 검색어를 바로 떠올리기 어렵다.
- MealTube는 이때 `먹방`, `예능`, `짧은 영상`, `편안한 분위기`와 같은 키워드 카드를 제시하여 사용자가 현재 원하는 콘텐츠의 방향을 선택하도록 한다.
- 이 과정에서 사용자의 막연한 요구는 여러 선택 데이터로 변환되고, 시스템은 이를 취향 프로필로 구성하여 벡터 검색에 활용한다.
- 따라서 카드 선택은 단순한 UI 요소가 아니라, 사용자가 원하는 방향을 추천 과정에 직접 입력하고 그 결과에 반영되도록 만드는 과정이다.

**3-3. 핵심 흐름**

```
카드 선택
→ 긍정 / 부정 선택 데이터 저장
→ 취향 프로필 생성
→ 프로필 텍스트 임베딩
→ Vector / Hybrid Search 수행
→ 추천 결과 정렬
→ 추천 근거 표시
```

예를 들어 사용자가 다음과 같이 선택했다고 가정할 수 있다.

| 구분 | 선택 예시 |
| --- | --- |
| 긍정 선택 | 먹방, 예능, 짧은 영상 |
| 부정 선택 | 공포, 긴 다큐 |

이 경우 시스템은 사용자의 취향을 다음과 같이 해석할 수 있다.

> “밥을 먹으며 가볍게 볼 수 있는 짧은 예능 또는 먹방 영상을 선호함.”
> 

이렇게 생성된 취향 프로필은 단순 문장으로 끝나는 것이 아니라, 임베딩을 거쳐 벡터 검색 질의로 변환된다.

## 4. 키워드 검색과 벡터 검색

**4-1. 키워드 검색이란?**

- 키워드 검색은 사용자가 입력한 단어가 콘텐츠의 제목, 설명, 태그 등에 포함되어 있는지를 기준으로 결과를 찾는 방식이다.
- 예를 들어 `공포`라고 검색하면 `공포 영상`, `공포 영화 리뷰`, `공포 썰`처럼 해당 단어가 포함된 콘텐츠가 주로 검색된다.
- 이 방식은 단순하고 직관적이며, 정확한 명칭이나 태그를 알고 있을 때는 효과적이다.
- 하지만 표현이 달라지면 관련 콘텐츠를 놓칠 수 있다.
- 예를 들어 `공포`라는 단어가 직접 들어가지 않은 `실화 괴담`, `미제 사건`, `스릴러 리뷰`, `범죄 심리 분석` 같은 콘텐츠는 관련성이 높아도 우선순위에서 밀릴 수 있다.

**4-2. 벡터 검색이란?**

- 벡터 검색은 텍스트나 콘텐츠 설명을 숫자 벡터로 변환한 뒤, 벡터 간의 거리를 기준으로 의미적으로 가까운 콘텐츠를 찾는 방식이다.
- 사람은 `공포`, `괴담`, `스릴러`, `미제 사건`이 비슷한 분위기를 가진 단어라는 것을 쉽게 이해할 수 있다.
- 하지만 컴퓨터는 이 단어들을 그대로 보면 서로 다른 문자열로 인식한다.
- 따라서 문장이나 단어를 컴퓨터가 비교할 수 있는 숫자 형태로 바꾸는 과정이 필요하다.
- 이 과정을 **임베딩**이라고 한다.
- 임베딩된 벡터들은 의미가 비슷할수록 가까운 위치에 놓인다.
- 벡터 검색은 사용자 질의 벡터와 콘텐츠 벡터가 얼마나 가까운지 비교하여 관련성이 높은 콘텐츠를 찾는다.

쉽게 말해 키워드 검색이 **“같은 단어가 들어 있는가?”**를 보는 방식이라면, 벡터 검색은 **“의미적으로 가까운가?”**를 보는 방식이다.

| 구분 | Keyword Search | Vector Search | Hybrid Search |
| --- | --- | --- | --- |
| 검색 기준 | 입력 단어의 직접 일치 | 의미적 유사성 | 키워드 일치 + 의미 유사성 |
| 강점 | 정확한 명칭·태그 검색에 강함 | 표현이 달라도 관련 의미 탐색 가능 | 정확성과 의미 탐색을 함께 활용 |
| 예시 | `공포`가 들어간 영상 검색 | `괴담`, `미제 사건`, `스릴러`까지 탐색 | 관련성이 높고 납득 가능한 결과 구성 |
| 한계 | 단어가 다르면 놓칠 수 있음 | 임베딩 품질에 따라 결과가 달라질 수 있음 | 점수 결합 방식 설계가 필요함 |

**4-3. 왜 콘텐츠 추천에 벡터 검색이 필요한가**

- 콘텐츠 추천에서는 사용자가 원하는 것을 항상 정확한 단어로 표현하지 못하는 경우가 많다.
- “재밌는 거”, “밥 먹을 때 볼만한 거”, “긴장감 있는 거”처럼 사용자의 요구는 막연한 경우가 많다.
- 키워드 검색은 이런 요구를 직접 처리하기 어렵지만, 벡터 검색은 선택 데이터나 문장 형태의 취향 정보를 의미 기반으로 검색에 활용할 수 있다.
- 따라서 MealTube에서는 사용자의 카드 선택 결과를 취향 프로필로 만들고, 이를 벡터 검색 질의로 변환한다.
- 이 과정을 통해 사용자가 직접 정확한 검색어를 만들지 않아도, 의미적으로 가까운 콘텐츠를 추천할 수 있다.

## 5. 설명 가능한 벡터 검색 기반 추천

**5-1. 설명 가능한 추천이 필요한 이유**

- 기존 추천 시스템은 결과를 제공하지만, 사용자는 그 결과가 왜 나왔는지 명확히 알기 어렵다.
- 추천 결과가 사용자의 취향과 맞더라도, 그 근거가 보이지 않으면 사용자는 결과를 우연처럼 받아들일 수 있다.
- MealTube는 추천 결과의 근거를 함께 보여주는 방향을 지향한다.
- 사용자는 자신이 선택한 카드가 어떤 방식으로 추천 결과에 반영되었는지 확인할 수 있다.
- 이 구조는 사용자가 추천 결과를 수동적으로 받아보는 방식에서 벗어나, 직접 선택 과정에 참여하고 결과를 이해할 수 있게 한다.

**5-2. 벡터 검색이 설명 가능성에 기여하는 방식**

- MealTube가 설명 가능한 추천 구조를 만들 수 있는 이유는 사용자의 선택 데이터를 벡터 검색 과정과 직접 연결하기 때문이다.
- 사용자가 선택한 카드는 단순한 선호 표시가 아니라, 추천 질의를 구성하는 명시적인 입력값이 된다.
- 예를 들어 사용자가 `공포`, `미스터리`, `범죄` 카드를 선택하면, 시스템은 이를 “긴장감 있고 몰입감 있는 미스터리 또는 범죄 관련 콘텐츠를 선호함”이라는 취향 프로필로 변환할 수 있다.
- 이 취향 프로필은 임베딩을 통해 벡터로 변환되고, 콘텐츠 벡터와 비교된다.
- 이후 유사도가 높은 콘텐츠가 추천 결과로 제시된다.
- 이 과정에서 시스템은 추천 이유를 다음과 같이 설명할 수 있다.

> 사용자가 선택한 `공포`, `미스터리`, `범죄` 키워드와 의미적으로 가까운 콘텐츠이기 때문에 추천되었습니다.
> 

또는 더 구체적으로 다음과 같이 설명할 수 있다.

> 이 영상은 제목에 `공포`라는 단어가 직접 포함되어 있지는 않지만, `괴담`, `미제 사건`, `범죄 실화` 태그를 포함하고 있어 사용자의 선택 키워드와 의미적으로 가깝습니다.
> 

**5-3. 열린 알고리즘과 사용자 개입**

- MealTube는 추천 과정을 완전히 숨기지 않는다.
- 사용자가 어떤 키워드를 선택했는지, 그 선택이 어떤 취향 프로필로 바뀌었는지, 어떤 검색 방식으로 결과가 나왔는지 보여줄 수 있다.
- 이 점에서 MealTube는 기존 블랙박스형 추천과 달리, 사용자가 추천 흐름을 시각적으로 확인할 수 있는 열린 추천 구조를 지향한다.
- 사용자는 단순히 알고리즘이 보여주는 결과를 기다리는 것이 아니라, 추천 전 단계에서 원하는 방향으로 직접 개입한다.
- 예를 들어 “짧고 가벼운 영상은 보고 싶지만, 공포나 긴 다큐는 피하고 싶다”는 요구를 카드 선택으로 명시할 수 있다.
- 시스템은 이 명시적 입력을 바탕으로 취향 프로필을 만들고, 벡터 검색을 통해 의미적으로 가까운 콘텐츠를 찾는다.
- 결과적으로 사용자는 자신의 취향에 맞는 콘텐츠를 더 빠르고 직관적으로 선택할 수 있다.
- 또한 추천 결과에 대한 근거까지 확인할 수 있기 때문에, 결과를 이해하고 납득하기도 쉬워진다.

## 6. Azure AI Search를 사용하는 이유

**6-1. 직접 추천 모델을 학습하는 방식의 부담**

- 추천 시스템을 직접 만들려면 많은 과정이 필요하다.
- 사용자 행동 데이터를 수집해야 하고, 콘텐츠 데이터셋을 구축해야 하며, 클릭률, 시청 시간, 좋아요, 이탈률 같은 데이터를 바탕으로 추천 모델을 학습해야 한다.
- 이후 모델 성능을 평가하고, 실제 서비스에 배포한 뒤에도 지속적으로 재학습과 운영 관리가 필요하다.
- 하지만 이번 프로젝트의 목적은 유튜브 수준의 대규모 추천 알고리즘을 구현하는 것이 아니다.
- 핵심은 사용자의 막연한 취향 선택을 검색 가능한 데이터로 바꾸고, 벡터 검색을 통해 의미적으로 가까운 콘텐츠를 찾는 과정을 보여주는 것이다.

**6-2. Azure AI Search의 역할**

- Azure AI Search는 애플리케이션에서 검색 기능을 구현할 수 있도록 도와주는 클라우드 기반 검색 서비스이다.
- 일반적인 텍스트 검색뿐만 아니라, 콘텐츠의 의미를 벡터로 표현하여 검색하는 Vector Search 구조로 확장할 수 있다.
- 또한 키워드 검색과 벡터 검색을 함께 활용하는 Hybrid Search 구조도 적용할 수 있다.
- MealTube에서는 콘텐츠 제목, 설명, 태그 같은 텍스트 데이터를 검색 대상으로 사용하고, 동시에 이 데이터를 임베딩 벡터로 변환하여 의미 기반 검색이 가능하도록 설계한다.
- 이를 통해 복잡한 추천 모델 학습보다, 검색 구조와 추천 로직 설계에 집중할 수 있다.

**6-3. MealTube에서의 적용 구조**

```
[카드 선택]
      ↓
[취향 프로필 생성]
      ↓
[Embedding 생성]
      ↓
[Azure AI Search Index]
- title / description / tags : 텍스트 검색용
- embedding                 : 벡터 검색용
      ↓
[Vector Search 또는 Hybrid Search]
      ↓
[부정 태그 패널티 적용]
      ↓
[추천 결과 + 추천 근거]
```

| 기술 포인트 | 설명 |
| --- | --- |
| Vector Field | 콘텐츠의 의미 벡터를 저장하는 필드 |
| Vector Query | 사용자 취향 프로필 벡터와 가까운 콘텐츠를 검색하는 질의 |
| Hybrid Search | 키워드 검색과 벡터 검색 결과를 함께 활용하는 방식 |
| Metadata / Tags | 추천 근거 표시와 부정 선택 반영에 사용하는 정보 |

## 7. 데모의 목적과 사용 기능

**7-1. 데모의 목적**

- 데모의 목적은 카드 UI를 보여주는 데서 끝나지 않는다.
- 핵심은 사용자의 카드 선택 결과가 어떻게 검색 질의로 변환되고, 검색 방식에 따라 추천 결과가 어떻게 달라지는지를 보여주는 것이다.
- 따라서 데모에서는 `선택 데이터 → 취향 프로필 → 임베딩 → Keyword / Vector / Hybrid Search → 추천 결과 → 추천 근거`의 흐름이 드러나야 한다.
- 특히 중요한 부분은 Keyword Search와 Vector Search의 결과 차이다.
- 예를 들어 사용자가 `공포`, `미스터리`, `범죄`를 선택했을 때 Keyword Search는 `공포`라는 단어가 직접 들어간 콘텐츠를 중심으로 결과를 보여줄 수 있다.
- 반면 Vector Search는 `괴담`, `스릴러`, `미제 사건`, `범죄 심리 분석`처럼 표현은 다르지만 의미적으로 가까운 콘텐츠를 찾을 수 있다.
- 이 차이를 데모에서 직접 보여주면, 벡터 검색이 단순 키워드 검색과 어떻게 다른지 직관적으로 전달할 수 있다.

## 10. 검색 결과 비교: Keyword / Vector / Hybrid

MealTube 데모에서 가장 중요한 장면은 동일한 사용자 선택 데이터를 기준으로 Keyword Search, Vector Search, Hybrid Search 결과를 비교하는 부분이다.

예를 들어 사용자가 다음과 같이 선택했다고 가정할 수 있다.

| 구분 | 선택 예시 |
| --- | --- |
| 긍정 선택 | 공포, 미스터리, 범죄 |
| 부정 선택 | 힐링, 로맨스 |

시스템은 이 선택을 바탕으로 다음과 같은 취향 프로필을 생성할 수 있다.

```
긴장감 있고 몰입감 있는 미스터리 또는 범죄 관련 콘텐츠를 선호함.
```

이때 검색 방식에 따라 결과는 다르게 나타날 수 있다.

| 검색 방식 | 결과 예시 | 해석 |
| --- | --- | --- |
| Keyword Search | 공포 영상 모음, 공포 영화 리뷰, 공포 썰 | 입력 키워드와 직접 일치하는 콘텐츠 중심 |
| Vector Search | 실화 괴담, 미제 사건 다큐, 범죄 심리 분석, 스릴러 리뷰 | 단어가 직접 일치하지 않아도 의미적으로 가까운 콘텐츠 탐색 |
| Hybrid Search | 공포 실화 사건, 미제 사건 리뷰, 범죄 다큐 요약 | 키워드 일치와 의미 유사성을 함께 반영 |

이 비교를 통해 사용자는 벡터 검색이 단순히 “검색어를 더 잘 찾는 기능”이 아니라, 사용자의 의도와 콘텐츠의 의미를 연결하는 검색 방식이라는 점을 이해할 수 있다.

## 11. 검색 결과 정렬과 추천 근거

검색 결과는 단순히 유사도만으로 보여주기보다, 키워드 일치 여부와 부정 선택 태그를 함께 고려하여 정렬할 수 있다.

예를 들어 최종 점수는 다음과 같은 방식으로 구성할 수 있다.

```
final_score
= vector_similarity
+ keyword_match_bonus
- negative_tag_penalty
```

각 요소의 의미는 다음과 같다.

| 요소 | 설명 |
| --- | --- |
| vector_similarity | 사용자 취향 벡터와 콘텐츠 벡터의 의미적 유사도 |
| keyword_match_bonus | 사용자가 선택한 키워드와 콘텐츠 태그가 직접 일치할 때 추가 점수 |
| negative_tag_penalty | 사용자가 부정 선택한 태그와 관련된 콘텐츠에 적용하는 감점 |

추천 근거는 막연하게 생성하지 않고, 실제 선택 데이터와 콘텐츠 태그를 기반으로 구성한다.

| 추천 콘텐츠 | 추천 근거 예시 |
| --- | --- |
| 편의점 신상 먹방 하이라이트 | 선택한 `먹방`, `짧은 영상` 태그와 일치 |
| 웃긴 예능 클립 모음 | 선택한 `예능`, `가벼움` 성향과 유사 |
| 공포 괴담 모음 | `공포`를 부정 선택하여 추천 순위 하락 |

이 구조를 사용하면 추천 결과가 단순히 “AI가 추천한 결과”처럼 보이지 않고, 어떤 선택과 어떤 기준이 반영되었는지 설명할 수 있다.

## 12. 정리

- MealTube는 기존 유튜브 알고리즘을 대체하려는 서비스가 아니라, 사용자가 유튜브 추천이나 검색 과정에서 영상을 바로 고르지 못하는 순간을 보완하는 서비스이다.
- 기존 유튜브 추천은 사용자 입장에서 추천 이유가 명확히 보이지 않는 블랙박스형 추천에 가까울 수 있다.
- MealTube는 사용자가 직접 키워드 카드를 선택하고, 그 선택을 취향 프로필과 벡터 검색 질의로 변환한다.
- 따라서 사용자는 알고리즘에 취향을 일방적으로 맡기는 것이 아니라, 원하는 방향을 명시적으로 입력하며 추천 과정에 직접 개입할 수 있다.
- MealTube는 선택 키워드, 관련 태그, 유사도, 추천 근거를 함께 보여주는 설명 가능한 추천 방식을 지향한다.
- 기존 키워드 검색은 사용자가 입력한 단어와 직접 일치하는 콘텐츠를 찾는 데 강하지만, 표현이 다른 관련 콘텐츠를 놓칠 수 있다.
- 반면 벡터 검색은 의미적 유사성을 기준으로 콘텐츠를 탐색할 수 있기 때문에 사용자의 막연한 취향이나 현재 상황을 추천 결과에 반영하기에 적합하다.
- 최종적으로 MealTube는 열린 알고리즘 구조를 통해 사용자가 자신의 취향에 맞는 콘텐츠를 더 빠르고 직관적으로 선택할 수 있도록 돕는 설명 가능한 벡터 검색 기반 콘텐츠 추천 시스템이다.
