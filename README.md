# MealTube(가명)

설명 가능한 벡터 검색 기반 콘텐츠 추천 시스템

## 프로젝트 소개

MealTube는 사용자가 콘텐츠 플랫폼에서 원하는 영상을 바로 고르기 어려울 때, 키워드 카드를 선택해 현재 취향을 입력하고 그 결과를 바탕으로 콘텐츠를 추천하는 서비스이다.

사용자는 직접 긴 검색어를 입력하지 않아도 `먹방`, `예능`, `짧은 영상`, `공포`, `다큐`, `경제`, `여행`과 같은 키워드 카드를 선택할 수 있다.

시스템은 선택된 키워드를 취향 프로필로 변환하고, 이를 검색 질의로 활용하여 의미적으로 가까운 콘텐츠를 추천한다.

MealTube의 목표는 기존 콘텐츠 추천 알고리즘을 대체하는 것보다, 사용자가 콘텐츠를 고르기 어려운 순간에 현재 취향을 더 명확하게 반영할 수 있도록 돕는 데 있다.

---

## 프로젝트 배경

유튜브, 넷플릭스, 쇼츠, 릴스처럼 콘텐츠 플랫폼은 이미 일상적으로 사용되고 있다. 콘텐츠의 양은 많아졌지만, 사용자는 오히려 무엇을 볼지 결정하기 어려운 상황을 자주 겪는다.

특히 밥을 먹을 때, 쉬는 시간, 이동 중처럼 가볍게 콘텐츠를 보고 싶은 상황에서는 사용자가 원하는 것을 정확한 검색어로 표현하기 어렵다.

예를 들어 다음과 같은 요구는 명확한 검색어로 바꾸기 애매하다.

- 밥 먹을 때 가볍게 볼 영상
- 짧고 재밌는 콘텐츠
- 지금 기분에 맞는 영상
- 너무 무겁지 않은 예능 클립
- 공포는 싫지만 긴장감 있는 콘텐츠

기존 플랫폼에서는 홈 화면 추천 목록을 계속 스크롤하거나, 새로고침하거나, 직접 검색어를 입력해야 한다.

MealTube는 이 과정에서 사용자가 자신의 현재 취향을 직접 입력하고, 그 선택이 추천 결과에 반영되도록 하는 것을 목표로 한다.

## 검색 방식

MealTube는 콘텐츠 검색을 위해 Keyword Search, Vector Search, Hybrid Search를 함께 고려한다.

| 검색 방식 | 설명 |
| --- | --- |
| Keyword Search | 사용자가 선택한 키워드와 직접 일치하는 콘텐츠 검색 |
| Vector Search | 표현이 달라도 의미적으로 가까운 콘텐츠 검색 |
| Hybrid Search | 키워드 일치와 의미 유사성을 함께 반영 |

예를 들어 사용자가 `공포`, `미스터리`, `범죄`를 선택한 경우 검색 방식에 따라 결과가 달라질 수 있다.

| 검색 방식 | 결과 예시 | 특징 |
| --- | --- | --- |
| Keyword Search | 공포 영상, 공포 영화 리뷰, 공포 썰 | 입력 키워드와 직접 일치하는 콘텐츠 중심 |
| Vector Search | 실화 괴담, 미제 사건, 스릴러 리뷰, 범죄 심리 분석 | 단어가 직접 일치하지 않아도 의미적으로 가까운 콘텐츠 탐색 |
| Hybrid Search | 공포 실화 사건, 미제 사건 리뷰, 범죄 다큐 요약 | 키워드 일치와 의미 유사성을 함께 반영 |

## 추천 점수 구조

추천 결과는 단순 유사도만으로 정렬하지 않고, 키워드 일치 여부와 부정 선택 태그를 함께 고려한다.

final_score

= vector_similarity

- keyword_match_bonus
- negative_tag_penalty

| 요소 | 설명 |
| --- | --- |
| vector_similarity | 사용자 취향 벡터와 콘텐츠 벡터 간의 의미적 유사도 |
| keyword_match_bonus | 선택 키워드와 콘텐츠 태그가 직접 일치할 때 부여하는 추가 점수 |
| negative_tag_penalty | 사용자가 피하고 싶은 태그와 관련된 콘텐츠에 적용하는 감점 |

이 구조를 통해 사용자의 명시적인 선택을 추천 결과에 반영할 수 있다.

## 추천 근거 표시

MealTube는 추천 결과와 함께 추천 근거를 표시하는 것을 목표로 한다.

추천 근거는 다음 데이터를 기반으로 구성된다.

- 사용자가 선택한 긍정 키워드
- 사용자가 선택한 부정 키워드
- 콘텐츠 제목
- 콘텐츠 설명
- 콘텐츠 태그
- 키워드 일치 여부
- 벡터 유사도
- 최종 추천 점수

예시는 다음과 같다.

| 추천 콘텐츠 | 추천 근거 예시 |
| --- | --- |
| 편의점 신상 먹방 하이라이트 | 선택한 `먹방`, `짧은 영상` 태그와 일치 |
| 웃긴 예능 클립 모음 | 선택한 `예능`, `가벼움` 성향과 유사 |
| 공포 괴담 모음 | `공포`를 부정 선택하여 추천 순위 하락 |

추천 근거를 함께 제공하면 사용자는 어떤 선택과 기준이 추천 결과에 반영되었는지 확인할 수 있다.

## Azure AI Search를 사용하는 이유

MealTube의 목적은 유튜브 수준의 대규모 추천 모델을 직접 학습하는 것이 아니다.

핵심은 사용자의 선택 데이터를 검색 가능한 형태로 변환하고, 의미적으로 가까운 콘텐츠를 찾는 구조를 구현하는 것이다.

추천 시스템을 직접 구현하려면 사용자 행동 데이터 수집, 콘텐츠 데이터셋 구축, 클릭률·시청 시간·좋아요·이탈률 기반 모델 학습, 성능 평가, 배포, 재학습 과정이 필요하다.

이번 프로젝트에서는 추천 모델 자체를 학습하기보다, 사용자의 선택 데이터를 검색 질의로 변환하고 Azure AI Search를 통해 콘텐츠를 탐색하는 구조에 집중한다.

Azure AI Search를 사용하면 다음 기능을 활용할 수 있다.

- 텍스트 기반 키워드 검색
- 벡터 필드를 활용한 의미 기반 검색
- 키워드 검색과 벡터 검색을 결합한 하이브리드 검색
- 콘텐츠 메타데이터와 태그 기반 필터링
- 검색 점수를 활용한 추천 결과 정렬

## 데모 계획

데모에서는 사용자의 카드 선택 결과가 검색 질의로 변환되고, 검색 방식에 따라 추천 결과가 어떻게 달라지는지를 보여줄 예정이다.

데모 흐름은 다음과 같다.

1. 사용자가 긍정 / 부정 키워드 카드를 선택한다.
2. 선택 데이터를 바탕으로 취향 프로필을 생성한다.
3. 취향 프로필을 임베딩하여 검색 질의로 변환한다.
4. Keyword Search, Vector Search, Hybrid Search 결과를 비교한다.
5. 최종 점수 계산 방식을 적용해 추천 결과를 정렬한다.
6. 각 추천 결과에 대한 추천 근거를 표시한다.

데모의 핵심은 카드 UI 자체보다, 사용자의 선택 데이터가 검색 질의로 변환되고 검색 방식에 따라 결과가 달라지는 과정을 보여주는 것이다.

## 최종 목표

MealTube는 사용자가 콘텐츠를 고르기 어려운 상황에서 현재 취향을 직접 입력하고, 그 선택이 반영된 추천 결과를 받을 수 있도록 돕는 콘텐츠 추천 시스템이다.

최종적으로는 다음 세 가지를 구현하는 것을 목표로 한다.

1. 사용자의 막연한 요구를 키워드 선택 데이터로 변환
2. 선택 데이터를 취향 프로필과 벡터 검색 질의로 변환
3. 추천 결과와 추천 근거를 함께 제공

이를 통해 사용자는 자신의 취향에 맞는 콘텐츠를 더 빠르고 직관적으로 선택할 수 있다.
